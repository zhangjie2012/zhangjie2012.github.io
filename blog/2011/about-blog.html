2012.08.09

嗯，你看到的没错，右侧下方的确是广告，今天新添加的。

找广告的原因是我想做一些尝试，为以后的新站做准备，本站不是为了赚钱，只是技术上的积累和分享。看过很多站，导致我很烦广告，相信大家的感觉和我一样，尤其是闪烁和弹出广告，看着让人心烦。因此，我没有找那种广告。我选择的是C++的书籍，毫不夸张的说，那些广告是我"精心"选择的。本来打算少找几本的，后来发现太少的话，不成学习系统。一共九本书，无一不是经典中的经典。我有其中的8本，没有《C++程序设计语言》，看过其中的7本，《深度探索C++模型》没看完。剩下的书籍，平均看过两边，《STL源码剖析》看过3遍，《Effective STL》看过一遍，其他的都是两遍。我这么说不是为了炫耀，这没什么可炫耀的。我只想让大家知道这些书我不是随随便便推荐给大家的，我很负责任的告诉大家它们很棒。

我学习C++的时候走了很多弯路，尤其是看书这一块，经验告诉我，按照我所列的书籍列表阅读是一个很不错的选择，我推荐给大家。 还有很多好书，比如《C++沉思录》、《C++编程思想》等，数据结构方面的书、软件工程、编码风格，我没列。全列出来太长了，等我会自己写WP模板以后，再想一个好的方案去解决吧，现在无能为力。

很久了，我想写一篇关于如何学习C++的文章，但是迟迟不敢下手。想写的原因是看着很多人走弯路乱读书、很浮躁冒进、受VC++6.0等非标准的毒害、受国内书籍的毒害等等很不爽，就好像N多人没学 template 去学 STL、更甚者去看源码；不敢下手的原因是自己 C++ 资历尚浅，如果言语不当，实在是害人。再想想看吧！

2011.11.13

一位 ID 为 Leeyou 的网友评我的 MFC框架技术简单研讨 问我为什么不在 sina sea 上弄个博客或者弄个个人 Wiki？其实我以前也想过弄个个人博客，但是有几方面的顾虑：

  1. 我对网站开发方面一点都不懂，怎么管理？
  2. 如果有个人博客之后，之前的博客怎么办？扔掉？还是搬过去。从 csdn 博客搬到博客园，大部分文章已经不复存在了;
  3. 博客园的好处在于人多，便于交流；个人博客呢？

经过我几天的考虑之后，我给自己找了些理由来开通个人博客：

  1. 我问了一些有个人博客的网友。说当下有很多博客模板可以使用，管理博客可用 wordpress，很方便。如果想要做的好一点的话，可以学学 php 。我恍然大悟，之前我一直在做桌面应用软件，没有涉及到网站方面。以为很复杂，"没文化，好可怕";
  2. 对于之前的博客，我确实有些不舍，因为毕竟做了一年了。但是细细想来，csdn 、新浪博客、微薄、博客园、人人网……，每一样都不愿放弃。事实上，每一样都没有做好。我博客园的副标题是："佛曰：忘记并不等于从未存在，一切自在来源于选择，而不是刻意。不如放手，放下的越多，越觉得拥有的更多。"现在博客的副标题依然是这个。贴在哪里好久，却没有自己品味它的含义。试想假如放弃csdn、新浪博客、博客园博客，把精力全部放在一个博客上，是不是会好一点呢？是不是真的放下的越多，拥有的更多呢？
  3. 相信假如博客质量高一些，坚持的时间长一些的话。访问量应该不成问题;
  4. 刚拿到项目报酬，基金上也没有问题了。

剩下的就是一些选型了：

  1. 博客的域名选取。我想过一些博客域名，但都放弃了。我一直想我想要什么要的博客？那就是"简单"，言简意赅、大道至简。这和我个人性格有很大的关系，如果你看过我的博客园博客，你应该就会明白了。简单，丝毫不加浮夸的成分。我写代码，乃至做人做事，都是这个原则，不喜复杂。简单 for simple。
    • www.simple.com 好像已经被占用了;
    • www.moresimple.com 倒是没有没有被占用，但是 more 过于强调，不喜;
    • www.keepsimple.com www.keepsimple.org 这个不错，也被占用了。
    • 将 www.keepsimply.org 作为域名，simply for "简单的，朴素的"也不错(形容词，副词等语法角度没有考虑，这方面我是白丁)。
  2. 博客的定位。我之前的博客是讲技术和生活感悟分开的，这个博客的定位是 "技术+生活感悟"。生活感悟，我将分一个"思绪飘零"类，其他的都是技术文章。
  3. 博客的内容。博客园博客会继续用，不过将"Direct3D初涉"系列完成之后，就不会再添加文章了，查东西的时候可以用的。博客均为原创，不会转载文章，而且整理部分也会减少。
  4. 文章风格。文章开头表明发布时间；正文字体为"宋体+consolas"，无首行缩进；文章末尾表明参考资源。
  5. 文章版本控制。直接续写，表明续写时间或修改时间。
